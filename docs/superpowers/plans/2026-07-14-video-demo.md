# Cours d'Arabe Video Demo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a silent, English-text, 27-second HyperFrames product demo showing the real Cours d'Arabe interface and its core learning flow.

**Architecture:** Capture the locally served static application into immutable image assets, then compose six deterministic HyperFrames scenes around those captures. A root composition controls scene timing and transitions; each beat remains an isolated HTML composition with its own registered GSAP timeline.

**Tech Stack:** HyperFrames CLI, HTML5, CSS, GSAP 3.14.2, local Python HTTP server, Chromium capture, H.264 MP4.

## Global Constraints

- Output format: 1920×1080 landscape at 30 fps.
- Target duration: 27 seconds; hard maximum: 30 seconds.
- Language: English overlays; authentic Arabic content remains visible.
- Audio: none; no narration, music, or sound effects.
- Palette and fonts must come from captured tokens: Inter, Noto Naskh Arabic, `#06090f`, `#0c1119`, `#131b28`, `#f8fafc`, `#94a3b8`, `#f0b429`, `#f7c948`, `#34d399`, `#a78bfa`, `#fb923c`.
- Motion must be deterministic: no `Math.random()`, `Date.now()`, asynchronous timeline construction, or infinite repeats.
- Every scene element needs an entrance; transitions own non-final exits.
- No repository commits are possible because `/home/ta/Documents/العربية/cours arabe` is not a Git repository. Each task ends with a filesystem or CLI verification checkpoint.
